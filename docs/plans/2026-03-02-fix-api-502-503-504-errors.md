# Fix API 502/503/504 Errors Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Fix all 502 (relay unreachable), 503 (relay not configured / oref), and 504 (GDELT timeout) errors surfaced in the browser console so the app loads data correctly.

**Architecture:** The app uses a Vercel edge layer (`api/*.js`) that proxies requests to a Railway relay server (`scripts/ais-relay.cjs`). The relay itself calls upstream APIs (AISStream, OpenSky, Polymarket, Telegram, NASA GDELT, etc.). Some calls — EONET and GDELT — go directly from the Vercel server layer. All 502s indicate the relay is either down, misconfigured, or not reachable from Vercel. The plan fixes connectivity first, then improves resilience for when the relay is unavailable.

**Tech Stack:** Node.js (CJS) on Railway, Vercel Edge Functions (JS), TypeScript server handlers, Upstash Redis for caching.

## Background: Error Map

| Error | Endpoint | Root Cause |
|---|---|---|
| 502 | `/api/ais-snapshot` | Relay unreachable or `WS_RELAY_URL` not set |
| 502 | `/api/polymarket` | Same relay |
| 502 | `/api/telegram-feed` | Same relay |
| 502 | `/api/rss-proxy` | Same relay (for `RELAY_ONLY_DOMAINS`) |
| 503 | `/api/oref-alerts` | Relay down; fallback fires correctly (silent failure is OK) |
| 504 | `/api/intelligence/v1/search-gdelt-documents` | GDELT upstream timeout (30s default; needs shorter cutoff) |
| 503 | `eonet.gsfc.nasa.gov` | Direct NASA call from browser failing; needs relay or fallback |
| misc | News digest | Downstream of relay being down; will self-heal once relay is up |

## Task 3: Add stale-if-error cache extension to relay endpoints

**Problem:** When the relay is temporarily down, all relay-backed endpoints return hard 502s with no grace period. Browsers can't serve stale cached data.

**Files:**
- Modify: `api/ais-snapshot.js`
- Modify: `api/polymarket.js`
- Modify: `api/telegram-feed.js`
- Modify: `api/opensky.js`

**Step 1: Update `api/ais-snapshot.js` to extend stale-if-error**

```javascript
// api/ais-snapshot.js — increase stale-if-error from 900s to 3600s
import { createRelayHandler } from './_relay.js';

export const config = { runtime: 'edge' };

export default createRelayHandler({
  relayPath: '/ais/snapshot',
  timeout: 12000,
  requireApiKey: true,
  requireRateLimit: true,
  cacheHeaders: (ok) => ({
    'Cache-Control': ok
      ? 'public, max-age=60, s-maxage=300, stale-while-revalidate=600, stale-if-error=3600'
      : 'public, max-age=10, s-maxage=30, stale-while-revalidate=120',
    ...(ok && { 'CDN-Cache-Control': 'public, s-maxage=300, stale-while-revalidate=600, stale-if-error=3600' }),
  }),
});
```

**Step 2: Create `api/relay-health.js`**

```javascript
import { getCorsHeaders } from './_cors.js';
import { getRelayBaseUrl, getRelayHeaders, fetchWithTimeout } from './_relay.js';

export const config = { runtime: 'edge' };

export default async function handler(req) {
  const corsHeaders = getCorsHeaders(req, 'GET, OPTIONS');

  if (req.method === 'OPTIONS') {
    return new Response(null, { status: 204, headers: corsHeaders });
  }

  const relayBaseUrl = getRelayBaseUrl();
  if (!relayBaseUrl) {
    return new Response(JSON.stringify({ relay: 'not_configured', details: 'WS_RELAY_URL not set' }), {
      status: 503,
      headers: { 'Content-Type': 'application/json', ...corsHeaders },
    });
  }

  const start = Date.now();
  try {
    const response = await fetchWithTimeout(
      `${relayBaseUrl}/health`,
      { headers: getRelayHeaders() },
      5000,
    );
    const latencyMs = Date.now() - start;
    if (response.ok) {
      const body = await response.json().catch(() => ({}));
      return new Response(JSON.stringify({ relay: 'ok', latencyMs, ...body }), {
        status: 200,
        headers: { 'Content-Type': 'application/json', 'Cache-Control': 'no-store', ...corsHeaders },
      });
    }
    return new Response(JSON.stringify({ relay: 'error', status: response.status, latencyMs }), {
      status: 502,
      headers: { 'Content-Type': 'application/json', 'Cache-Control': 'no-store', ...corsHeaders },
    });
  } catch (error) {
    return new Response(JSON.stringify({ relay: 'error', details: error?.message || String(error) }), {
      status: 502,
      headers: { 'Content-Type': 'application/json', 'Cache-Control': 'no-store', ...corsHeaders },
    });
  }
}
```
